## المراحل

### م-1 · الأساس + لوحة إدارة القوالب  ← نبدأ هنا
- نماذج البيانات: `DocTemplate` · `TemplateSection` (6 أنواع أقسام) · `TemplateElement` (15 نوع عنصر) + الأنواع المعدودة.
- ثوابت الموديول: أنواع المستندات الـ5، أنواع العناصر الـ15 (تسمية/أيقونة/فئة/قابلية التكرار)، أنواع الأقسام، وباني هيكل أوليّ لكل نوع مستند.
- بذور `INITIAL_TEMPLATES` + `TemplatesStore` (create/duplicate/update/archive/restore/delete + audit) + تسجيل الحفظ IndexedDB.
- لوحة القوالب: بطاقات + فلترة (نوع/حالة) + بحث + إجراءات (تعديل/نسخ/أرشفة/استعادة/حذف) + معاينة سريعة + مودال «قالب جديد».
- جذع صفحة المحرّر (route جاهز يعرض الهيكل للقراءة + ملاحظة «المرحلة 2»).
- التوجيه + رابط التنقّل «🧩 مولّد القوالب» + اختبارات المتجر.

### م-2 · المحرّر البنائي (MVP بالقوائم)
قائمة الأقسام والعناصر: إضافة/حذف/نسخ/إخفاء/ترتيب (↑↓) + لوحة خصائص حسب نوع العنصر + معاينة حية + حفظ (تحديث/إصدار جديد).

### م-3 · الهياكل الأولية الغنية + كل أنواع العناصر الـ15
هيكل جاهز مناسب لكل نوع مستند (عرض سعر/فاتورة/اتفاقية/أمر دفع/ورق رسمي) + خصائص كل عنصر كاملة.

### م-4 · إنشاء مستند من قالب
نموذج ديناميكي من القالب (حقول إلزامية/شرطية/محسوبة) → معاينة → توليد وتصدير PDF عبر بنية `helpers/export|documents` الحالية + حفظ نسخة في المستندات.

### م-5 · الهوية البصرية على القالب
تطبيق ألوان/خط/شعار (يُبنى فوق `SettingsStore.docBranding` و`TemplateEditor` الحاليين) على القالب بنقرة + معاينة.

### م-6 · ميزات الذكاء (محاكاة)
`templatesAI.ts`: مصمّم ألوان · كاتب نصوص · مولّد قوالب بوصف · مدقّق قبل الحفظ — بواجهة موحّدة جاهزة لاستبدالها بخادم.

### م-7 · (لاحقاً/اختياري) ترقية المحرّر إلى WYSIWYG سحب وإفلات
كانفاس + سحب وإفلات + ترتيب مرئي — بعد استقرار الـMVP.
